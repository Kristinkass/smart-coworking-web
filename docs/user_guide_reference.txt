Введение
В данном документе представлено руководство пользователя системы для анализа и обработки речевых сигналов.
Настоящее руководство содержит информацию о назначении, условиях применения, подготовительных действиях и операциях, которые доступны пользователю при работе с данным приложением.
Данный документ разработан в соответствии с требованиями ГОСТ Р 59795-2021 «Информационные технологии. Комплекс стандартов на автоматизированные системы».
Область применения
Программное обеспечение для анализа и обработки речевых сигналов предназначено для использования в следующих областях деятельности:
лингвистические исследования и фонетический анализ;
криминалистика и голосовая идентификация;
медицинская диагностика на основе анализа речи;
разработка систем распознавания речи;
научные исследования в области акустики;
образовательная деятельность при обучении иностранным языкам.
Краткое описание возможностей
Пользователю представлены следующее возможности:
импорт и воспроизведение аудиозаписей в форматах WAV, MP3, FLAC, OGG, M4A;
автоматическая оценка качества аудиосигнала при загрузке (громкость, уровень шума);
автоматическая фонемная сегментация аудио при помощи нейросети Wav2Vec 2.0;
автоматическое распознавание речи и получение текстовой расшифровки движком Vosk;
отображение фонем в режимах кириллицы (Русский) и международного алфавита (IPA);
ручное редактирование временных границ фонем, изменение их меток, добавление и удаление;
поиск звуковых сочетаний по лингвистическим шаблонам и пользовательским маскам;
экспорт результатов в форматы TXT, CSV, PDF, а также TextGrid и PitchTier;
сохранение проектов во встроенную базу данных для последующей работы.
Уровень подготовки пользователя
Уровень подготовки пользователя – начальный. Пользователь может взаимодействовать с приложением, не имея специальных навыков в обработке аудиосигналов. Достаточно иметь базовые навыки пользования персональным компьютером, включая управление мышью и работу с приложениями Windows.
Для работы с результатами анализа рекомендуется иметь базовые знания в области фонетики и лингвистики.
Перечень эксплуатационной документации, с которыми необходимо ознакомиться пользователю
Пользователю необходимо ознакомиться со следующей документацией:
Руководство пользователя;
Справка в приложении (встроенная система помощи).
Назначение и условия применения
Виды деятельности, функции, для автоматизации которых предназначено данное средство автоматизации
Основной функцией приложения является автоматизация процессов анализа и обработки речевых сигналов, включая:
автоматическую сегментацию речи на фонетические единицы (фонемы) и определение их временных границ;
автоматическую текстовую расшифровку (транскрибирование) речевого сигнала;
визуализацию спектрально-временных характеристик звука (построение осциллограмм и спектрограмм);
быстрый контекстный поиск фонемных сочетаний по лингвистическим шаблонам;
формирование отчетов и экспорт результатов во внешние специализированные системы.

Подготовка к работе
Состав и содержание дистрибутивного носителя данных
Дистрибутив приложения содержит:
исходный код приложения;
программные модули и системные библиотеки для обработки аудио и инференса нейросетей;
ресурсы приложения (интерфейсные иконки, шрифты, демонстрационный проект);
шаблон структуры локальной базы данных SQLite;
руководство пользователя (эксплуатационную документацию).
Порядок загрузки данных программы и установки программы
Установка приложения:
распаковать архив с исходным кодом приложения в рабочую директорию на диске;
установить на компьютер интерпретатор Python версии 3.10 или выше;
открыть командную строку (терминал) в папке с распакованным исходным кодом;
выполнить команду установки необходимых библиотек:
pip install -r requirements.txt
дождаться завершения загрузки и установки всех зависимых пакетов.
Запуск приложения:
открыть командную строку (терминал) в папке проекта;
выполнить команду запуска приложения:
python main.py
дождаться автоматической проверки целостности моделей (при первом запуске будет произведена их асинхронная загрузка) и открытия стартового окна.
Порядок проверки работоспособности
Для проверки корректной установки и работоспособности приложения выполнить следующие действия:
запустить приложение (см. подразд. 3.2);
проверить открытие стартового экрана с кнопкой «Загрузить проект» и списком ранее созданных проектов;
загрузить один из примеров проектов из списка;
проверить отображение спектрограммы для оригинального аудио;
нажать кнопку «Воспроизведение» и убедиться в наличии звука;
закрыть проект и вернуться на стартовый экран (нажать кнопку «Назад»).
Если все этапы пройдены успешно, приложение готово к работе.
Описание операций
Данный раздел настоящего руководства пользователя содержит описание выполняемых функций, задач и процедур программой.
Общее описание технологического процесса
Типичный процесс работы с приложением включает следующие этапы:
запуск приложения и выбор проекта;
загрузка аудиофайла в новый проект;
анализ речи с автоматической сегментацией на фонемы;
визуализация результатов на спектрограмме;
поиск необходимых звуковых сочетаний;
экспорт результатов анализа;
сохранение проекта для последующей работы.
Описание экранных форм и интерфейса
Стартовый экран
При запуске приложения открывается стартовый экран, представленный на рисунке 4.2.1.
Рисунок 4.2.1 – Стартовый экран
Данное окно обеспечивает доступ к локальной библиотеке проектов и позволяет создавать новые исследования, открывать ранее сохранённые проекты, выполнять их поиск по названию и удалять ненужные записи.
Главное рабочее окно
После выбора или создания проекта открывается главное рабочее окно, представленное на рисунке 4.2.2.
Рисунок 4.2.2 – Главное рабочее окно
Центральную часть интерфейса занимает графическая область визуализации, содержащая синхронизированные осциллограмму и спектрограмму аудиозаписи. Поверх спектрограммы отображается фонемная сетка, сформированная на основе результатов автоматической сегментации. Каждая фонема представлена отдельным временным интервалом с указанием её обозначения. Для повышения удобства редактирования границы сегментов отображаются в виде интерактивных вертикальных маркеров, которые пользователь может перемещать с помощью мыши.
В процессе воспроизведения аудиозаписи положение текущего фрагмента отображается специальным курсором, синхронно перемещающимся по всем графическим областям. Под областью визуализации размещены элементы управления воспроизведением, включая кнопку запуска и остановки аудио, регулятор громкости и индикатор текущего времени.
Под графиками представлена область, предназначенная для выполнения поисковых и аналитических операций. Здесь располагаются элементы задания поискового шаблона, таблица найденных фонемных сочетаний и текстовое поле, отображающее распознанную речевую транскрипцию. Выбор любого найденного результата автоматически переводит курсор воспроизведения к соответствующему участку аудиозаписи, что обеспечивает быстрый переход к интересующему пользователя фрагменту.
Для редактирования результатов автоматической сегментации реализован механизм интерактивного изменения фонем. При двойном щелчке по выбранному сегменту открывается диалоговое окно фонетической клавиатуры (рисунок 4.2.3 и 4.2.4), содержащее набор доступных фонемных обозначений. После выбора нового символа соответствующая метка на графике обновляется, а изменения автоматически сохраняются во внутреннем представлении проекта.

Рисунок 4.2.4 – Экранная форма клавиатуры с IPA-фонемами
Для сохранения результатов анализа предусмотрено отдельное окно настройки экспорта. Окно экспорта представлено на рисунке 4.2.5.
Рисунок 4.2.5 – Экранная форма окна настройки экспорта
Через данный интерфейс пользователь может выбрать формат выходного файла и параметры выгрузки данных. Поддерживается экспорт в текстовые форматы TXT и CSV, генерация отчётов в формате PDF, а также формирование файлов TextGrid и PitchTier.
Описание экранных форм и интерфейса
Создание нового проекта
Подготовительные действия:
Находиться на стартовом экране приложения.
Выполнение операции:
Нажать зеленую кнопку «+ Создать новый проект» в правой части окна;
В открывшемся системном проводнике выбрать исходный аудиофайл форматов WAV, MP3, FLAC, OGG или M4A;
В появившемся диалоговом окне ввода текста указать понятное название для создаваемого проекта и нажать кнопку «ОК»;
В следующем окне выбора каталогов указать папку на жестком диске, внутри которой программа создаст персональную директорию исследования.
Ожидаемый результат:
В выбранном каталоге создается папка проекта, исходный аудиофайл копируется внутрь неё под именем audio.<расширение>. Стартовое окно закрывается, открывается главное рабочее окно проекта с построенными графиками осциллограммы и спектрограммы.
Открытие существующего проекта
Подготовительные действия:
Находиться на стартовом экране приложения.
Выполнение операции:
Вариант 1: найти проект в списке «Мои проекты» в левой части окна (при необходимости использовать строку поиска над списком) и дважды кликнуть по нему левой кнопкой мыши;
Вариант 2: нажать кнопку «Открыть проект из папки» в нижнем правом углу, в диалоговом окне проводника выбрать папку проекта системы «Фоноскоп» на диске и подтвердить выбор.
Ожидаемый результат:
Открывается главное рабочее окно проекта. Из базы данных SQLite полностью восстанавливается ранее сохраненная разметка фонем, текст расшифровки и результаты последнего поиска.
Управление воспроизведением звукового сигнала
Подготовительные действия:
Проект должен быть загружен в главное рабочее окно.
Выполнение операции:
Нажать кнопку «Play» на панели инструментов под графиками для запуска проигрывания аудио;
Во время воспроизведения нажать ту же кнопку (изменившую название на «Pause») для приостановки;
Для перемещения по временной шкале зажать ЛКМ на движке слайдера времени и перетащить его в нужную сторону, либо совершить одиночный клик в произвольное место полосы прокрутки;
Для регулировки громкости переместить слайдер «Громкость».
Ожидаемый результат:
Аудиосигнал воспроизводится в динамиках. Желтый вертикальный маркер перемещается по графикам осциллограммы и спектрограммы синхронно со звуком. Цифровой индикатор времени корректно обновляет показания.
Автоматический нейросетевой анализ аудиозаписи
Подготовительные действия:
Проект должен быть загружен в главное рабочее окно.
Выполнение операции:
Нажать комбинацию горячих клавиш Ctrl+R или выбрать в верхнем меню «Проект» → «Повторно выровнять»;
В диалоговом окне предупреждения подтвердить перезапись текущих данных разметки, нажав «Да»;
Дождаться окончания процесса анализа, наблюдая за движением шкалы прогресс-бара в нижней части окна.
Ожидаемый результат:
Нейросетевое ядро строит фонемную разметку и накладывает сетку вертикальных маркеров на спектрограмму. В нижнем текстовом поле автоматически отображается распознанный движком Vosk текст аудиозаписи.
Интерактивное ручное редактирование фонемной разметки
Подготовительные действия:
В главном окне должна отображаться фонемная сетка.
Выполнение операции:
Смещение границ: навести курсор мыши на вертикальную границу фонемы (линия изменит цвет). Зажать ЛКМ и переместить ее по шкале времени, после чего отпустить кнопку мыши.
Изменение фонемы: дважды щелкнуть ЛКМ внутри фонемного сегмента. В открывшемся окне виртуальной фонетической клавиатуры выбрать нужный символ и нажать на него.
Добавление сегмента: кликнуть ПКМ по области спектрограммы, в контекстном меню выбрать пункт «Добавить здесь».
Удаление сегмента: кликнуть ПКМ по спектрограмме, в контекстном меню выбрать «Удалить этот сегмент».
Отмена ошибки: нажать сочетание клавиш Ctrl+Z для отмены последнего действия.
Ожидаемый результат:
Границы фонем плавно сдвигаются. Значения букв на графике мгновенно обновляются. Все ручные изменения сохраняются во временной памяти приложения и готовы к записи в базу данных.
Поиск фонемных сочетаний
Подготовительные действия:
В главном окне должна отображаться фонемная сетка.
Выполнение операции:
Поиск по шаблону: выбрать из выпадающего списка предустановленный шаблон (например, СЃС — согласный-ударный гласный-согласный) и нажать кнопку «Найти».
Поиск по маске: выбрать в выпадающем списке «Пользовательский...», нажать на поле ввода. В открывшемся окне клавиатуры последовательно выбрать интересующие звуки (например, [п][р][и]), закрыть клавиатуру и нажать кнопку «Найти»
Ожидаемый результат:
На экране отображаются все найденные фонемы с указанием их типа. Пользователь может быстро найти интересующие его фонетические единицы.
Сохранение проекта и экспорт результатов
Подготовительные действия:
В проект внесены изменения, или выполнен поиск созвучий.
Выполнение операции:
Для сохранения проекта в базу данных SQLite нажать Ctrl+S или выбрать в меню «Проект» → «Сохранить проект»;
Для экспорта результатов выбрать в меню «Проект» → «Экспорт...» (или нажать Ctrl+E);
В открывшемся диалоговом окне экспорта выбрать желаемую вкладку (TXT/CSV, PDF или спецформаты TextGrid/PitchTier);
Задать необходимые настройки (например, экспортировать всю разметку или только отмеченные результаты поиска), указать имя файла и нажать кнопку «Экспортировать».
Ожидаемый результат:
При сохранении данные записываются в БД SQLite и дублируются в JSON-файл в папке проекта. При экспорте по указанному пути на жестком диске создается структурированный файл выбранного формата.
Аварийные ситуации
В данном разделе настоящего руководства пользователя содержится информация о действиях пользователя в случаях возникновения аварийных ситуаций.
Действия при ошибках загрузки аудиофайла
Возможные ошибки:
«Ошибка чтения аудиофайла» (файл поврежден или заблокирован другой программой);
«Файл не поддерживается» (выбран неподдерживаемый формат);
Предупреждения о качестве: «Тихий сигнал» (RMS < 0.005) или «Повышенный уровень шума».
Действия:
Убедиться, что загружаемый файл находится в одном из поддерживаемых форматов (WAV, MP3, FLAC, OGG, M4A);
Проверить целостность аудиофайла (попробовать открыть его в другом приложении для воспроизведения);
При получении предупреждения о слишком тихой записи («Тихий сигнал») усилить уровень сигнала в стороннем аудиоредакторе и повторить импорт;
Попробовать перезагрузить приложение и повторить загрузку файла.
Действия при зависании приложения
При полном зависании приложения:
Нажать комбинацию клавиш Ctrl+Alt+Delete для открытия диспетчера задач;
Найти в списке процессов «python» (или «[ф]оноскоп»);
Выделить процесс и нажать кнопку «Снять задачу»;
Перезапустить приложение из командной строки командой python main.py.
При медленной работе:
Закрыть другие запущенные приложения;
Проверить объём свободной памяти на диске (должно быть не менее 3 ГБ);
Перезагрузить компьютер;
Если проблема сохраняется, обратиться к системному администратору.
Действия при потере данных
При случайном удалении результатов анализа:
Восстановить данные разметки из резервного файла phonemes.json, который автоматически дублируется в локальной папке проекта при каждом сохранении;
Если проект не был сохранен пользователем (сочетанием клавиш Ctrl+S или через меню), данные текущей сессии восстановить невозможно. Рекомендуется регулярно сохранять работу.
Действия при отказе оборудования
При отказе дисплея:
Перезагрузить компьютер;
Проверить подключение монитора;
Убедиться в исправности видеокарты.
При отказе динамиков при воспроизведении:
Проверить громкость в системе (значок динамика на панели задач);
Убедиться в подключении динамиков;
Проверить значок звука в приложении;
Обновить драйверы звуковой карты.
Действия в других аварийных ситуациях
В случае возникновения других аварийных ситуаций при работе с программой необходимо выполнить следующее:
определить природу и причины аварийной ситуации;
принять меры для предотвращения распространения аварийной ситуации и минимизации возможного ущерба;
сообщить о произошедшем разработчику и заинтересованным сторонам;
восстановить работоспособность системы и возобновить технологический процесс.
Рекомендации по освоению
Для успешной работы с приложением пользователю рекомендуется:
Начать с демонстрационного проекта – открыть встроенный проект «ДЕМО: Пример разметки» для ознакомления с интерфейсом и функциями поиска без необходимости первичной обработки аудиозаписей.
Изучить встроенную справку – использовать клавишу F1 или меню «Справка» для быстрого вызова интерактивного руководства пользователя.
Использовать качественные файлы – для первых исследований выбирать короткие (до 30 секунд) аудиозаписи без сильного фонового шума.
Контролировать качество аудио – обращать внимание на автоматические предупреждения системы об уровне громкости и шума при загрузке аудиофайлов.
Регулярно сохранять результаты – использовать сочетание клавиш Ctrl+S после каждого этапа ручной корректировки маркеров фонем.
Использовать разные раскладки фонем – переключать режимы отображения фонем («Русский» / «IPA») и тестировать шаблоны поиска для быстрого обнаружения созвучий.
Рекомендации по использованию
Оптимизация работы приложения
Качество аудиозаписи: для получения максимальной точности распознавания нейросетью используйте одноканальные записи с частотой дискретизации 16 кГц без выраженного фонового шума.
Использование GPU-ускорения: при анализе длинных аудиофайлов (более 10 минут) запускайте программу на компьютерах с видеокартами NVIDIA и поддержкой технологии CUDA, что ускоряет расчет в несколько раз.
Контроль оперативной памяти: при пакетной обработке большого количества файлов периодически закрывайте неиспользуемые проекты для освобождения ОЗУ.
Работа с результатами анализа
Ручная верификация: всегда визуально контролируйте результаты автоматической сегментации нейросети на спектрограмме и при необходимости вручную корректируйте границы маркеров.
Интеграция с профессиональным лингвистическим обеспечением: используйте экспорт в формат TextGrid для проведения профессионального спектрального анализа и интонационных исследований в средах Praat или ELAN.
Подготовка отчетности: экспортируйте результаты исследований в формат PDF для получения готовых отчетов с таблицами фонем и статистикой.
