Аннотация
Настоящая Программа и методика испытаний (ПМИ) предназначена для проверки выполнения заданных функций системы анализа и обработки речевых сигналов, проверки соответствия количественных и качественных характеристик, выявления и устранения недостатков в действиях и в разработанной документации на этапе проведения приёмочных испытаний.
Программа и методика испытаний разработана в соответствии с требованиями ГОСТ Р 59795‑2021 «Информационные технологии. Комплекс стандартов на автоматизированные системы».

Объект испытаний
Наименование системы
Система для анализа и обработки речевых сигналов – специализированная компьютерная система, предназначенная для автоматизации процессов сегментации, идентификации фонетических структур и проведения спектрального анализа.
Комплектность АС
Программное обеспечение поставляется в виде каталога со следующими компонентами:
Исполняемые файлы:
main.py – основной файл запуска приложения;
requirements.txt – список зависимостей Python.
Программные модули:
src/engine.py – вычислительное ядро (нейросетевой анализ аудио);
src/model_downloader.py – асинхронный загрузчик моделей;
src/database.py – модуль работы с базой данных проектов;
src/search_engine.py – модуль поиска фонетических сочетаний;
src/export_manager.py – модуль экспорта результатов;
src/gui/ – пакет графического интерфейса пользователя.
Модели нейросетей:
models/phonoscopic/ – предобученная модель Wav2Vec2 для фонемного анализа;
models/vosk/ – модель распознавания речи Vosk (опционально).
Данные и документация:
data/projects.db – файл базы данных SQLite (создаётся при первом запуске);
demo_project/ – демонстрационный проект с примером разметки;
встроенная справка (вызов по F1).
Системные зависимости:
интерпретатор Python 3.14;
библиотеки: PyTorch, Transformers, PyQt6, Librosa, Vosk (полный список в requirements.txt);
CUDA Toolkit (опционально, для ускорения на GPU).
Цель испытаний
Целью испытаний является проверить соответствие программного обеспечения требованиям, предъявляемым в техническом задании, а именно:
Функциональные требования:
автоматическая загрузка моделей и проверка целостности:
проверка наличия моделей;
быстрая проверка критических файлов;
глубокая проверка целостности;
загрузка фонемной модели;
загрузка STT модели;
импорт аудиофайлов в стандартных форматах (WAV, MP3, FLAC, OGG);
воспроизведение аудиозаписей внутри приложения с элементами управления (старт, пауза, перемотка, регулировка громкости);
визуализация осциллограммы и спектрограммы загруженного аудиосигнала;
автоматическая сегментация аудио на фонемы с использованием нейросетевой модели Wav2Vec2;
отображение фонемной разметки в двух режимах: международный фонетический алфавит (IPA) и кириллическая транскрипция;
ручное редактирование границ фонем (перетаскивание мышью) и их значений (выбор через экранную клавиатуру);
добавление и удаление отдельных фонемных сегментов через контекстное меню;
определение ударных и безударных гласных, согласных фонем;
текстовая расшифровка аудиозаписи с использованием движка Vosk;
поиск звуковых сочетаний по предустановленным шаблонам (СГС, Г, С и другие) или по пользовательской маске вида [ф][о][н];
отображение результатов поиска в табличном виде с временными метками и возможностью отметки для экспорта;
выделение найденных сочетаний на спектрограмме цветными областями;
сохранение и загрузка проектов с восстановлением состояния поиска и выбранных для экспорта элементов;
экспорт результатов анализа в текстовые форматы (TXT, CSV), формат PDF, а также в специализированные форматы TextGrid и PitchTier для программы Praat.
Требования к показателям назначения:
время полного запуска приложения не должно превышать 60 секунд;
время загрузки аудиофайла длительностью до 10 секунд и построения спектрограммы не должно превышать 3 секунд;
максимальная поддерживаемая длительность анализируемого сигнала – до 60 минут;
время выполнения полного анализа (сегментация на фонемы и текстовая расшифровка) для файла длительностью до 60 минут не должно превышать разумного времени ожидания и должно сопровождаться индикацией прогресса.
Требования к надёжности и безопасности:
корректная обработка ошибок;
сохранение целостности данных;
обеспечение конфиденциальности обрабатываемых данных (вся обработка выполняется локально, без передачи данных по сети);
стабильная работа приложения при длительной эксплуатации без утечек памяти.

Объём испытаний
Перечень этапов испытаний и проверок
В процессе проведения приёмочных испытаний должно быть протестировано ПО для анализа и обработки речевых сигналов.
Приемочные испытания включают проверку:
установка и запуск – проверка запуска приложения из исходного кода или собранного исполняемого файла;
функциональные тесты – проверка всех основных функций;
тесты производительности – проверка времени отклика;
тесты надёжности – проверка обработки ошибок;
тесты совместимости – проверка на различных конфигурациях;
документирование результатов – формирование отчётов.
Последовательность проведения
Испытания проводятся в последовательности, указанной в перечне этапов.
Требования по испытаниям программных средств
тестирование должно проводиться на минимальной конфигурации ПК (4 ГБ ОЗУ, Windows 10, без GPU);
тестирование должно проводиться на рекомендуемой конфигурации (8 ГБ ОЗУ, Windows 10/11, с GPU NVIDIA);
все обнаруженные ошибки должны быть задокументированы с указанием шагов для воспроизведения;
все функции должны быть протестированы не менее чем в двух сценариях;
при обнаружении критических ошибок – остановка испытаний до исправления.
Перечень работ, проводимых после завершения испытаний
После завершения испытаний формируется протокол испытаний с описанием проведенных проверок и их результатов. В случае обнаружения несоответствий программа дорабатывается разработчиком.
Условия и порядок проведения испытаний
Испытания проводятся в учебной лаборатории на компьютерах с операционной системой Windows 11. Программа устанавливается на компьютеры в соответствии с инструкциями из руководства пользователя.
Проверки выполняются в соответствии с перечнем этапов испытаний студентом-исполнителем в присутствии преподавателя-заказчика. Результаты фиксируются в протоколе.
Материально-техническое обеспечение испытаний
Приёмочные испытания проводятся на программно-аппаратном комплексе Заказчика в следующей минимальной конфигурации:
процессор с тактовой частотой не менее 2 ГГц, не менее 2 ядер;
оперативная память не менее 4 ГБ (для режима работы без GPU) или 8 ГБ (для режима работы с GPU);
видеокарта NVIDIA с поддержкой CUDA версии 11.0 или выше (опционально, при отсутствии видеокарты анализ выполняется на центральном процессоре);
монитор;
мышь или тачпад;
клавиатура;
наушники или колонки;
операционная система Windows 11.
Метрологическое обеспечение испытаний
Программа испытаний не требует использования специализированного измерительного оборудования.
Отчётность
Результаты испытаний ПО, предусмотренные настоящей программой, фиксируются в протоколах, содержащих следующие разделы:
назначение испытаний и номер раздела требований ТЗ на разработку, по которому проводят испытание;
состав технических и программных средств, используемых при испытаниях;
указание методик, в соответствии с которыми проводились испытания, обработка и оценка результатов;
условия проведения испытаний и характеристики исходных данных;
средства хранения и условия доступа к тестирующей программе;
обобщённые результаты испытаний;
выводы и рекомендации по доработке (если применимо).
Этап проведения предварительных испытаний завершается оформлением «Акта предварительных и приемочных испытаний программного обеспечения для анализа и обработки речевых сигналов».

1
Запуск приложения
Запуск файла main.py через интерпретатор Python или запуск скомпилированного исполняемого файла. Ожидание загрузки стартового окна.
Приложение успешно установлено и запущено, стартовый экран отображён, время загрузки не более 60 секунд
2
Создание нового проекта
Нажатие кнопки «Создать новый проект», выбор аудиофайла формата WAV через диалог выбора файла, ввод названия проекта, выбор папки для сохранения, подтверждение создания.
Проект успешно создан, аудиофайл скопирован в папку проекта, открыто главное окно с осциллограммой и спектрограммой. Время загрузки файла длительностью до 10 секунд не превышает 3 секунд.
3
Импорт аудиофайлов различных форматов
Аналогично пункту 2 выполнить создание проектов с аудиофайлами форматов MP3, FLAC, OGG.
Все файлы успешно загружены, осциллограмма и спектрограмма отображаются корректно, ошибок не возникает.
4
Воспроизведение аудиозаписи
В главном окне нажать кнопку Play, проверить воспроизведение звука. Нажать Pause для паузы. Повторно нажать Play для продолжения. Переместить ползунок времени в другую позицию и проверить воспроизведение с новой позиции.
Звук воспроизводится корректно, кнопки управления работают, временная метка на осциллограмме и спектрограмме перемещается синхронно с воспроизведением. Текущее и общее время отображаются корректно.
5
Регулировка громкости
Перемещение ползунка громкости в главном окне, проверка изменения громкости при воспроизведении.
Громкость воспроизведения изменяется в соответствии с положением ползунка.
6
Масштабирование графиков
В главном окне использовать колесо мыши для увеличения и уменьшения масштаба осциллограммы и спектрограммы. Перетаскивание графика мышью для перемещения по временной шкале.
Масштаб графиков изменяется плавно, перемещение по временной шкале работает корректно. При сильном уменьшении масштаба текст фонем скрывается автоматически.
7
Автоматический анализ аудио
В главном окне с загруженным аудиофайлом выбрать в меню «Проект» пункт «Повторно выровнять» или нажать сочетание клавиш Ctrl+R. Подтвердить запуск анализа в диалоговом окне. Дождаться завершения процесса, наблюдая за индикатором прогресса.
На осциллограмме и спектрограмме отображаются границы фонем и их метки. Индикатор прогресса показывает этапы выполнения анализа. Время выполнения анализа зависит от длительности файла и аппаратной конфигурации.
8
Текстовая расшифровка аудио
После завершения автоматического анализа проверить наличие текстовой расшифровки в нижней панели главного окна в группе «Текстовая расшифровка».
В поле текстовой расшифровки отображается текст, распознанный системой Vosk. Текст соответствует содержанию аудиозаписи в разумных пределах точности распознавания.
9
Переключение режима отображения фонем
В меню «Вид» выбрать подменю «Вид фонем» и переключиться между режимами «Русский» и «IPA». Проверить изменение отображаемых меток фонем.
При выборе режима «Русский» фонемы отображаются в кириллической транскрипции. При выборе режима «IPA» фонемы отображаются символами международного фонетического алфавита.
10
Классификация фонем
Визуально проверить отображение фонем на спектрограмме. Ударные гласные должны отображаться с символом ударения, безударные гласные без него, согласные соответствующими символами.
Фонемы классифицированы корректно. Ударные гласные (а, о, у, ы, э, и) отображаются со знаком ударения, безударные без знака ударения.
11
Ручное редактирование границ фонем
Навести курсор мыши на вертикальную линию границы фонемы, зажать левую кнопку мыши и перетащить границу в новую позицию. Отпустить кнопку. Проверить, что соседние фонемы изменили свои границы соответствующим образом.
Граница фонемы перемещается плавно. При отпускании кнопки мыши временные метки соседних фонем обновляются. Метка фонемы перемещается в центр нового интервала.
12
Редактирование метки фонемы
Дважды щёлкнуть левой кнопкой мыши по фонемному региону. В открывшемся диалоговом окне экранной клавиатуры выбрать новую фонему. Нажать кнопку выбора.
Метка фонемы изменяется на выбранную. Отображение на спектрограмме обновляется.
13
Добавление и удаление фонем
Щёлкнуть правой кнопкой мыши по фонемному региону или границе. В контекстном меню выбрать «Добавить здесь». Проверить появление новой фонемы. Затем щёлкнуть правой кнопкой мыши по фонемному региону и выбрать «Удалить этот сегмент». Проверить удаление.
Новая фонема добавляется в указанном месте с меткой по умолчанию. При удалении фонема исчезает, соседние фонемы сдвигаются, занимая освободившееся время.
14
Отмена и повтор действий
Выполнить несколько операций редактирования разметки. Нажать Ctrl+Z для отмены последнего действия. Нажать Ctrl+Y для повтора отменённого действия.
Операции отменяются и повторяются корректно. Состояние разметки возвращается к предыдущему.
15
Поиск по предустановленным шаблонам
В панели поиска выбрать из выпадающего списка предустановленный шаблон, например «СГС (согл-удар.гласн-согл)». Нажать кнопку «Найти».
В таблице результатов отображаются все найденные сочетания, соответствующие шаблону. Для каждого результата указаны временные метки начала и конца, а также последовательность фонем.
16
Поиск по пользовательской маске
В панели поиска выбрать пункт «Пользовательский». Нажать на поле ввода для вызова экранной клавиатуры. Ввести последовательность фонем в формате, например, [п][р][и][в][е][т]. Нажать кнопку «Найти».
В таблице результатов отображаются все найденные вхождения заданной последовательности фонем.
17
Воспроизведение результата поиска
В таблице результатов поиска щёлкнуть левой кнопкой мыши по строке с найденным сочетанием.
Воспроизведение аудио начинается с момента начала выбранного сочетания и автоматически останавливается в момент его окончания. Область графика масштабируется к выбранному фрагменту.
18
Отметка результатов для экспорта
В таблице результатов поиска установить флажки в колонке «Экспорт» для нескольких найденных сочетаний. Нажать кнопку «Выбрать все» и проверить установку всех флажков. Нажать кнопку «Снять выделение» и проверить снятие всех флажков.
Флажки устанавливаются и снимаются корректно. Выбранные сочетания сохраняются в состоянии проекта.
19
Экспорт в текстовые форматы
В меню «Проект» выбрать «Экспорт». В открывшемся диалоговом окне на вкладке «Текстовые форматы» выбрать формат TXT или CSV. Указать параметры экспорта. Выбрать путь для сохранения файла и нажать «Экспортировать».
Файл экспорта создан по указанному пути. Содержимое файла соответствует выбранным параметрам и содержит данные о фонемах, текстовой расшифровке и отмеченных результатах поиска.
20
Экспорт в формат PDF
В диалоговом окне экспорта перейти на вкладку «PDF». Выбрать параметры содержимого отчёта. Указать путь для сохранения и нажать «Экспортировать».
PDF-файл создан по указанному пути. Документ открывается в программе просмотра PDF, содержит заголовок, таблицу фонем, текстовую расшифровку и отмеченные результаты поиска.
21
Экспорт в специализированные форматы
В диалоговом окне экспорта перейти на вкладку «Спецформаты». Выбрать формат TextGrid или PitchTier. Указать, что экспортировать: все фонемы или только отмеченные результаты поиска. Указать путь для сохранения и нажать «Экспортировать».
Файл в формате TextGrid или PitchTier создан по указанному пути. Файл открывается в лингвистической программе, например, Praat или ELAN, разметка соответствует данным проекта.
22
Сохранение проекта
В меню «Проект» выбрать «Сохранить проект» или нажать Ctrl+S. Проверить появление сообщения об успешном сохранении.
В папке проекта созданы или обновлены файлы phonemes.json и transcript.txt. В базе данных обновлена запись о проекте.
23
Сохранение проекта как нового
В меню «Проект» выбрать «Сохранить как». Ввести новое название проекта, выбрать папку для сохранения. Подтвердить создание.
Создана копия проекта в указанной папке с новым названием. В базе данных создана новая запись. Текущий проект переключён на новую копию.
24
Загрузка существующего проекта
Закрыть текущий проект через меню «Проект» и «Выход» или закрыть главное окно. В стартовом окне выбрать ранее созданный проект из списка и дважды щёлкнуть по нему.
Проект успешно загружен. Осциллограмма и спектрограмма отображаются. Фонемная разметка восстановлена. Текстовая расшифровка загружена. Состояние поиска и отмеченные для экспорта результаты восстановлены.
25
Удаление проекта из списка
В стартовом окне выбрать проект из списка и нажать кнопку «Удалить проект». Подтвердить удаление в диалоговом окне.
Проект удалён из списка в стартовом окне. Запись о проекте удалена из базы данных. Файлы проекта на диске сохранены.
26
Открытие демонстрационного проекта
При первом запуске программы или в списке проектов найти проект с названием «ДЕМО: Пример разметки». Открыть его двойным щелчком.
Демонстрационный проект успешно загружен. Отображается пример разметки с фонемами, доступны все функции анализа и поиска.
27
Отображение справки
В главном окне нажать клавишу F1 или выбрать в меню «Справка» пункт «Справка».
Открывается диалоговое окно с руководством пользователя. Справка содержит разделы по основным функциям программы.
28
Обработка ошибок при загрузке повреждённого файла
Попытаться создать новый проект и загрузить повреждённый или не являющийся аудиофайлом файл с расширением WAV.
Приложение выводит сообщение об ошибке загрузки аудиофайла. Приложение не завершается аварийно и остаётся в рабочем состоянии.
29
Производительность при анализе длинных файлов
Загрузить аудиофайл длительностью от 30 до 60 минут. Запустить автоматический анализ. Наблюдать за индикатором прогресса и потреблением оперативной памяти.
Анализ выполняется по фрагментам, индикатор прогресса отображает текущий этап. Потребление оперативной памяти остаётся в допустимых пределах и не растёт неограниченно. Приложение не зависает и остаётся отзывчивым.
30
Надёжность при длительной работе
Выполнить циклически несколько операций: загрузка проекта, запуск анализа, поиск сочетаний, экспорт результатов, сохранение проекта. Повторить цикл не менее пяти раз без перезапуска приложения.
Приложение стабильно работает на протяжении всех циклов. Ошибок, зависаний и утечек памяти не обнаружено.
