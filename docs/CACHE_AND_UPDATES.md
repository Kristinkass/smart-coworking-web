# Кэш, обновления и «где что лежит» — шпаргалка

Кратко: что обновляется само, что нужно перезагрузить вручную, и почему иногда «вижу старое».

---

## 1. Два вида «кэша» (не путать)

| Что | Где | Когда сбрасывается |
|-----|-----|-------------------|
| **Кэш браузера** | Телефон / Chrome / Safari | После жёсткого обновления страницы или смены `?v=` у скрипта |
| **Кэш сервера (layout в памяти)** | Python-процесс Flask | После `reload_layout()` или **перезапуска приложения** |

Браузер может держать **старый JS/CSS**, даже если на сервере уже новый код.  
Сервер может держать **старый layout.json в RAM**, даже если файл на диске уже изменён (обычно сбрасывается при сохранении через API).

---

## 2. Версии скриптов `?v=…`

В шаблонах JS подключается так:

```html
<script src=".../map_updated.js?v=29"></script>
```

**Правило:** изменили файл JS → **увеличьте число `v`** в шаблоне, который его подключает.

| Страница | Шаблон | Основные скрипты |
|----------|--------|------------------|
| Карта бронирования `/mapp` | `templates/mapp.html` | `map_updated.js`, `booking.js`, `places-list.js`, … |
| Редактор планировки | `templates/editor.html` | `editor/app.js`, `editor/geometry.js` |
| Остальные | `templates/base.html` | `main.js`, `input-limits.js` |

**Текущие версии на `/mapp` (проверяйте в `mapp.html`):**

- `booking.js?v=11`
- `map_updated.js?v=29`
- `places-list.js?v=6`
- `place-display.js?v=2`

> CSS и стили **внутри** `mapp.html` (блок `<style>`) **не имеют** `?v=`.  
> После правки inline-CSS достаточно обновить страницу; на телефоне — жёсткое обновление (см. ниже).

## 3. Как обновить у себя (браузер / телефон)

### Компьютер
- **Windows:** `Ctrl + F5` или `Ctrl + Shift + R`
- **Mac:** `Cmd + Shift + R`

### Телефон
- Закрыть вкладку → открыть `/mapp` заново  
- Или: настройки браузера → «Очистить данные сайта» (только для вашего домена)
- Safari iOS: иногда помогает режим «Частный доступ» для проверки

### Признак «вижу старую версию»
- Время бронирования снова «съехало на 3 часа»
- Старые цвета служебных зон
- Старый выбор времени (циферблат вместо select)

→ Почти всегда **не подтянулся новый JS**. Проверьте `?v=` и сделайте жёсткое обновление.

---

## 4. Когда перезапускать сервер / Docker

| Изменили | Достаточно |
|----------|------------|
| Только `.js` / `.html` / `static/` | Перезапуск **не обязателен** (Flask подхватит файлы; браузер — через `?v=` + F5) |
| `internal/*.py`, `requirements.txt`, `Dockerfile`, `.env` | **Перезапуск** приложения / `docker compose up --build` |
| `TZ=Europe/Moscow` в Docker | **Пересборка и перезапуск** контейнера |
| `static/layout.json` **вручную** (не через редактор) | Перезапуск **желателен** или любое сохранение места через API (вызовет `reload_layout`) |

## 5. Где хранятся данные (не кэш, но часто путают)

```
┌─────────────────────────────────────────────────────────┐
│  PostgreSQL (БД)                                        │
│  • пользователи, брони, тарифы, код места, статус       │
│  • НЕ хранит x/y/width/height столов                    │
└─────────────────────────────────────────────────────────┘
                          ↕ sync по code
┌─────────────────────────────────────────────────────────┐
│  static/layout.json                                     │
│  • координаты, стены, двери, container_code, rotation   │
│  • источник правды для геометрии карты                  │
└─────────────────────────────────────────────────────────┘
```

- **Бронирование** привязано к `place_id` в БД — смена **кода** места (`1A-T5` → `1A-T3`) историю броней **не ломает**.
- **Ссылки** вида `?place=1A-T26` после перенумерации кодов нужно обновить.

## 6. Время бронирования (чтобы не путать с кэшем)

| Источник | Для чего |
|----------|----------|
| **Браузер** (`new Date()`) | «Сегодня», текущее время, прошлые слоты на клиенте |
| **Сервер** (`Europe/Moscow`) | `is_past` в timegrid API |
| ~~`SERVER_NOW` в шаблоне~~ | **Удалён** — больше не используется |

Дата в форме на `/mapp` выставляется **JS при загрузке** (`initBookingDate()`), не с сервера.

Если время «опять +3 часа» после деплоя:
1. Жёсткое обновление страницы (`map_updated.js?v=29`).
2. Проверить TZ контейнера: `Europe/Moscow`.
3. Убедиться, что дата в форме — **сегодняшняя** по вашему локальному времени.

## 7. Коды мест (404, «не находит»)

**Частая причина:** в `layout.json` код с **кириллическими** буквами (`1РС-T26`), на экране выглядит как латиница (`1PC-T26`).

**Исправление (админ, POST):**
```http
POST /api/admin/places/reindex-codes
```
Перенумерует коды подряд (`1A-T1`, `1A-T2`, …) и чинит `location` у столов.

После перенумерации: **обновить страницу редактора** (F5).

## 8. Чеклист после ваших правок в коде

```
□ Изменил static/js/...  →  поднял ?v= в templates/mapp.html (или editor.html)
□ Изменил inline <style> в mapp.html  →  F5 на /mapp
□ Изменил internal/*.py  →  перезапустил сервер / Docker
□ Изменил Dockerfile / .env / requirements  →  docker compose up --build
□ Проверил на телефоне  →  закрыл вкладку, открыл заново
□ Менял коды мест  →  POST /api/admin/places/reindex-codes (если нужно)
```

## 10. Что **не** нужно делать

- Не править `layout.json` и БД вручную одновременно «на глаз» — используйте редактор или API.
- Не ожидать, что смена Python-кода подхватится без перезапуска процесса.
- Не забывать `?v=` — иначе пользователи месяцами сидят на старом JS (особенно на телефонах).
