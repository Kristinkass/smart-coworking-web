Общие положения
Настоящий документ содержит сведения, необходимые для работы программиста с системой для анализа и обработки речевых сигналов.
Наименование программы
Полное наименование системы: Система для анализа и обработки речевых сигналов.
Назначение и условия применения
Назначение программы
Основным назначением приложения является автоматизация процессов анализа и обработки речевых сигналов, включая:
автоматическую сегментацию речи на фонетические единицы (фонемы) и определение их временных границ;
автоматическую текстовую расшифровку (транскрибирование) речевого сигнала;
проведение спектрального анализа аудиозаписей;
быстрый контекстный поиск звуковых сочетаний по шаблонам;
формирование структурированных отчётов и результатов для дальнейшего исследования.
Функции, выполняемые программой
Перечень функций, реализуемых программой:
импорт и воспроизведение аудиофайлов форматов WAV, MP3, FLAC, OGG, M4A;
автоматический экспресс-анализ качества аудиозаписи при загрузке (уровень RMS, зашумленность ZCR, спектральная плоскостность);
автоматическая сегментация речи на фонемы с использованием нейросетевой модели Wav2Vec 2.0;
автоматическая текстовая расшифровка аудиозаписи средствами офлайн-модели Vosk;
классификация фонем на гласные (с выделением ударных) и согласные звуки;
интерактивная визуализация осциллограмм и спектрограмм с поддержкой ручной корректировки границ и значений фонем;
поиск звуковых сочетаний по лингвистическим шаблонам и пользовательским маскам;
сохранение проектов во встроенную СУБД SQLite;
экспорт результатов в форматы TXT, CSV, PDF, TextGrid и PitchTier.

Рисунок 1 – Архитектурная схема функционирования системы
В структуре программного комплекса выделены следующие основные компоненты.
Компонент GUI (Графический интерфейс пользователя) реализован на базе фреймворка PyQt6 и библиотеки PyQtGraph. Отвечает за визуализацию осциллограмм, спектрограмм и интерактивных маркеров границ фонем, управление воспроизведением, ввод поисковых шаблонов и параметров экспорта. Получает «Данные Проектов» из локального хранилища, принимает «Результат Анализа» от аналитического ядра для мгновенного отображения фонемной сетки, а также выводит «Статус Загрузки» (прогресс-бар) в процессе скачивания моделей.
Компонент Аналитическое Ядро (SpeechEngine) является вычислительным ядром приложения, написанное на Python с использованием библиотек PyTorch, Transformers и Librosa. Отвечает за математическую обработку звука (STFT), запуск инференса фонемной модели Wav2Vec 2.0 и распознавание речи Vosk. Запрашивает локальные «Веса Моделей» из хранилища данных, выполняет математические расчеты и возвращает структурированный «Результат Анализа» в GUI.
Компонент Автозагрузчик Моделей (model_downloader.py) представляет собой асинхронный модуль (класс DownloadWorker), функционирующий в фоновом потоке QThread. Отвечает за контроль наличия и целостности весов нейросетей при запуске. В случае отсутствия или повреждения файлов инициирует загрузку «Файлов фонемной модели» из репозитория Hugging Face Hub и «Файлов STT модели» с серверов Vosk по защищенному протоколу HTTPS. Передает скачанные веса в Хранилище данных, а динамику процесса (проценты, шаги распаковки) передает в виде «Статуса Загрузки» в интерфейс GUI.
Компонент Хранилище Данных (database.py и структура папок) объединяет в себе локальную реляционную базу данных SQLite (файл projects.db) и физическую файловую структуру каталогов на жестком диске. Обеспечивает долгосрочное хранение параметров проектов, истории поиска и настроек («Данные Проектов» для GUI), а также структурированное хранение и выдачу весов моделей для Аналитического ядра и Автозагрузчика.
Настройка аппаратного ускорения CUDA
Для выполнения нейросетевых вычислений (инференса фонемной модели Wav2Vec 2.0) на графическом процессоре (GPU) вместо CPU, программисту или системному администратору необходимо произвести настройку среды аппаратного ускорения NVIDIA CUDA.
Порядок настройки окружения:
Определение версии CUDA: Запустите командную строку Windows и выполните команду nvidia-smi. В правом верхнем углу выведенной таблицы найдите параметр CUDA Version. Это максимальная версия CUDA Toolkit, которую поддерживает установленный в системе видеодрайвер. При необходимости обновите драйвер видеокарты с сайта NVIDIA.
Установка NVIDIA CUDA Toolkit: Скачайте инсталлятор CUDA Toolkit стабильной версии (рекомендуется CUDA 12.1 или CUDA 11.8) из официального архива NVIDIA и выполните стандартную установку.
Установка PyTorch с поддержкой GPU: Для сопряжения установленной CUDA с фреймворком PyTorch необходимо сгенерировать точную команду установки на сайте pytorch.org. Откройте терминал в папке проекта и выполните команду установки (пример для версии CUDA 12.1): «pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121».
Установка остальных зависимостей: После успешного развертывания PyTorch GPU выполните установку остальных библиотек проекта: «pip install -r requirements.txt».
Верификация работы GPU: Запустите интерпретатор Python в терминале проекта и выполните проверочный код из Листинг 1.
Листинг 1 – Проверочный код CUDA
import torch
print("CUDA доступен:", torch.cuda.is_available())
if torch.cuda.is_available():
print("Активная видеокарта:", torch.cuda.get_device_name(0))
Получение значения «CUDA доступен: True» подтверждает успешную интеграцию аппаратного ускорения. Программа «Фоноскоп» при следующем запуске автоматически перенесет вычисления аналитического ядра на GPU.

Характеристика программных средств программы
Описание основных характеристик программы
Программное обеспечение «Фоноскоп» предназначено для автоматизированного анализа и разметки речевых сигналов. Система реализована как монолитное локальное Desktop-приложение, выполняющееся в рамках единого процесса Python и состоящее из следующих структурных компонентов:
Модуль GUI (интерфейс пользователя): реализован на PyQt6, отвечает за отрисовку синхронизированных графиков PyQtGraph, интерактивное редактирование и управление воспроизведением;
Аналитический модуль (SpeechEngine): осуществляет инференс нейросетевых моделей (Wav2Vec 2.0 и Vosk), цифровую обработку и экспресс-анализ качества аудиосигналов;
Локальное хранилище (database.py): база данных СУБД SQLite (projects.db) для долгосрочного структурированного хранения метаданных, истории поиска и результатов разметки;
Файловая система: локальные каталоги проектов для физического хранения аудиофайлов и резервных копий JSON-файлов разметки.
Временные характеристики программы
Среднее время запуска клиентского приложения (при наличии моделей) – не более 10 секунд.
Анализ короткого аудиофайла (до 20 секунд) занимает не более 5 секунд.
Время обмена данными между интерфейсом, базой данных SQLite и аналитическим ядром внутри единой оперативной памяти процесса Python является мгновенным и не превышает 0,01 секунды на операцию.
Режим работы программы
Программа функционирует в интерактивном режиме. Пользователь управляет проектами, запускает анализ и осуществляет ручную разметку через графический интерфейс (GUI), получая результаты в режиме реального времени. Все тяжелые вычисления (скачивание файлов, сегментация, транскрибирование) вынесены в асинхронные фоновые потоки QThread, что исключает зависание интерфейса.
Средства контроля правильности выполнения программы
Для обеспечения корректности функционирования реализованы следующие механизмы:
проверка формата, разрядности и целостности входных аудиофайлов средствами библиотек librosa и soundfile;
автоматическая валидация структуры таблиц базы данных SQLite при старте программы (метод create_tables);
быстрая (по критическим файлам) и глубокая (через тестовый инференс) проверка целостности нейросетевых моделей при каждом запуске;
вывод системных логов и сообщений об ошибках в консоль терминала и диалоговые окна QMessageBox.
Обращение к программе
Программа «Фоноскоп» запускается в интерактивном режиме с помощью вызова главного скрипта запуска main.py через интерпретатор Python.
Порядок вызова программы:
Запуск командной строки (системного терминала) в корневой папке приложения;
Выполнение команды вызова программы: python main.py;
Инициализация ядра: запускается инициализация главного класса Application. Проверяется наличие и целостность весов нейросетей. При обнаружении ошибок или отсутствии моделей открывается асинхронное окно загрузки ресурсов DownloadProgressWindow;
Инициализация СУБД: приложение подключается к локальной базе данных SQLite (projects.db). Из встроенной системной таблицы settings считываются глобальные параметры конфигурации (путь сохранения проектов по умолчанию, флаг автосохранения);
Инициализация GUI: инициализируется графическое окружение PyQt6, загружается и открывается стартовое окно StartWindow;
В процессе работы системные логи выполнения операций выводятся в консоль терминала, а сообщения об ошибках — в диалоговые окна QMessageBox и строку статуса главного окна.
Входные и выходные данные
Характер и организация входных данных
Входными данными являются цифровые аудиосигналы (аудиозаписи речи), характеризующиеся следующими параметрами:
Форматы файлов: WAV, MP3, FLAC, OGG, M4A;
Частота дискретизации: любая на входе. При импорте библиотека Librosa автоматически выполняет ресэмплинг (resampling) аудиосигнала до стандартной частоты 16 кГц (моно), необходимой для корректной работы нейросетевых моделей;
Разрядность (битность): 16 бит PCM (внутреннее представление при обработке тензоров в PyTorch);
Внутренняя структура: при загрузке аудиосигнал считывается в виде одномерного массива чисел (тип float32), представляющих изменение амплитуды во времени. В отношении массива производится автоматический экспресс-анализ качества звука по метрикам RMS (оценка громкости), ZCR (зашумленность) и Spectral Flatness.
Характер и организация выходных данных
Выходными данными являются результаты фонетического анализа и разметки речевых сигналов, экспортируемые в виде файлов следующих форматов.
Текстовые таблицы (TXT, CSV): содержат упорядоченный список фонем с указанием таймкодов начала, конца, длительности и статистический отчет о частотности каждого звука;
Документы PDF: готовые к печати отчеты, содержащие метаданные проекта, распознанный текст Vosk, таблицу фонем и результаты поиска созвучий;
Интервальные сетки (TextGrid): специализированный формат разметки. Алгоритм экспорта автоматически заполняет пустые интервалы (паузы) значением "" для сохранения непрерывности временной шкалы;
Контуры PitchTier: специализированный тональный формат точек частоты;
Системные логи: сообщения в консоли терминала о результатах проверки целостности моделей, инициализации CUDA/CPU и возникающих ошибках.
Сообщения
Сообщения в системе реализуются средствами графического интерфейса PyQt6 и делятся на следующие категории в зависимости от критичности событий:
Информационные сообщения (окна QMessageBox.information и строка status_lbl):
Уведомляют пользователя об успешном завершении операций.
Примеры: «Проект сохранен», «Фонемная модель готова!», «Анализ завершен».
Предупреждения (окна QMessageBox.warning):
Сообщают о некритичных проблемах, не останавливающих работу программы.
Примеры: Предупреждение системы качества звука о зашумленности («Повышенный уровень шума») или тихом сигнале.
Ошибки (окна QMessageBox.critical):
Информируют о невозможности продолжения текущей операции из-за неверных действий пользователя или сбоев среды.
Примеры: «Ошибка чтения аудиофайла», «Не удалось загрузить модели. Проверьте подключение к интернету».
Консольные логи (вывод в stdout терминала):
Предназначены для разработчика при отладке и сопровождении исходного кода. Выводятся в консоль запуска и содержат техническую информацию: статус инициализации CUDA, ход выполнения инференса Wav2Vec 2.0 по 30-секундным блокам, трассировки исключений Python (traceback) и параметры работы с СУБД SQLite.
Аварийные ситуации
В данном разделе настоящего руководства программиста содержится информация о действиях пользователя в случаях возникновения аварийных ситуаций.
Действия при ошибках загрузки аудиофайла
Возможные ошибки:
«Ошибка чтения аудиофайла» (файл поврежден или заблокирован другой программой);
«Файл не поддерживается» (выбран неподдерживаемый формат);
Предупреждения о качестве: «Тихий сигнал» (RMS < 0.005) или «Повышенный уровень шума».
Действия:
Убедиться, что загружаемый файл находится в одном из поддерживаемых форматов (WAV, MP3, FLAC, OGG, M4A);
Проверить целостность аудиофайла (попробовать открыть его в другом приложении для воспроизведения);
При получении предупреждения о слишком тихой записи («Тихий сигнал») усилить уровень сигнала в стороннем аудиоредакторе и повторить импорт;
Попробовать перезагрузить приложение и повторить загрузку файла.
Действия при зависании приложения
При полном зависании приложения:
Нажать комбинацию клавиш Ctrl+Alt+Delete для открытия диспетчера задач;
Найти в списке процессов «python» (или «[ф]оноскоп»);
Выделить процесс и нажать кнопку «Снять задачу»;
Перезапустить приложение из командной строки командой python main.py.
При медленной работе:
Закрыть другие запущенные приложения;
Проверить объём свободной памяти на диске (должно быть не менее 3 ГБ);
Перезагрузить компьютер.
Действия при потере данных
При случайном удалении результатов анализа:
Восстановить данные разметки из резервного файла phonemes.json, который автоматически дублируется в локальной папке проекта при каждом сохранении;
Если проект не был сохранен пользователем (сочетанием клавиш Ctrl+S или через меню), данные текущей сессии восстановить невозможно. Рекомендуется регулярно сохранять работу.
Действия при отказе оборудования
При отказе дисплея:
Перезагрузить компьютер;
Проверить подключение монитора;
Убедиться в исправности видеокарты.
При отказе динамиков при воспроизведении:
Проверить громкость в системе (значок динамика на панели задач);
Убедиться в подключении динамиков;
Проверить значок звука в приложении;
Обновить драйверы звуковой карты.
Действия в других аварийных ситуациях
В случае возникновения других аварийных ситуаций при работе с программой необходимо выполнить следующее:
определить природу и причины аварийной ситуации;
принять меры для предотвращения распространения аварийной ситуации и минимизации возможного ущерба;
сообщить о произошедшем разработчику и заинтересованным сторонам;
восстановить работоспособность системы и возобновить технологический процесс.
