# 3. Программная реализация

Программная реализация системы управления рабочими местами коворкинга выполнена в виде веб-приложения с клиент–серверной архитектурой. Серверная часть разработана на языке Python с использованием фреймворка Flask и СУБД PostgreSQL; клиентская часть представляет собой набор HTML-шаблонов, таблиц стилей и модулей JavaScript, исполняемых в браузере пользователя. Взаимодействие между клиентом и сервером организовано по двум каналам: серверный рендеринг страниц для типовых экранов и REST API для динамических операций — интерактивной карты, бронирования, редактора планировки и формирования отчётов.

При проектировании серверной части применён многослойный подход, где каждый уровень отвечает за свою зону ответственности. Обработчики HTTP-запросов принимают обращения от клиента и формируют ответ; сервисы содержат бизнес-логику бронирования и расчёта занятости; репозитории инкапсулируют доступ к базе данных; модели описывают предметную область в виде ORM-сущностей. Отдельный слой layout обслуживает геометрию планировки, хранящуюся в файле `layout.json`. Такой подход позволяет отделить транспортный уровень и отображение информации от правил её получения и обработки.

Отдельного внимания заслуживает разделение данных на две части. Пользователи, бронирования, тарифы, абонементы и расписание хранятся в PostgreSQL. Координаты мест, контуры стен и положение дверей — в `static/layout.json`. Связь между источниками осуществляется по коду места, что позволяет администратору редактировать планировку без изменения схемы базы данных при каждом перемещении объекта.

Статус на карте и доступность в временной сетке опираются на одни и те же правила занятости, реализованные в сервисном слое на сервере. Таким образом, визуализация пространства является центральным элементом пользовательского опыта: посетитель выбирает и бронирует место визуально на плане этажа, а не по абстрактному списку в таблице.
