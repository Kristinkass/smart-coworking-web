# Блок-схемы основных алгоритмов §2.4

Блок-схемы описывают логику программы простым языком, без имён переменных и фрагментов кода. Не дублируют BPMN: там — действия людей, здесь — автоматические проверки и расчёты.

Перенос в Word: [draw.io](https://app.diagrams.net/) или [mermaid.live](https://mermaid.live) → PNG.

---

## Рисунок 20 — Алгоритм проверки доступности и оформления бронирования

**Назначение:** проверить, можно ли забронировать место на выбранное время, и сохранить бронь при успехе.  
**Связь с BPMN:** шаги «проверить доступность» и «сохранить бронь» процесса «Создание бронирования».

```mermaid
flowchart TD
    START([Начало]) --> IN["Получить от пользователя:<br/>место, дату, время начала и окончания,<br/>число человек, тип тарифа"]
    IN --> S1{Коворкинг работает<br/>в выбранный день?}
    S1 -->|Нет| ERR1["Показать сообщение:<br/>коворкинг закрыт"]
    S1 -->|Да| S2{Дата и время<br/>указаны верно<br/>и не в прошлом?}
    S2 -->|Нет| ERR2["Показать сообщение:<br/>неверная дата или время"]
    S2 -->|Да| S3{Место не на<br/>техобслуживании?}
    S3 -->|Нет| ERR3["Показать сообщение:<br/>место недоступно"]
    S3 -->|Да| S4["Разбить выбранный интервал<br/>на отрезки по 15 минут"]
    S4 --> S5["Для каждого отрезка посчитать,<br/>сколько человек уже забронировано"]
    S5 --> S6{Во всех отрезках<br/>хватает свободных мест<br/>с учётом вместимости?}
    S6 -->|Нет| ERR4["Отказ: место занято<br/>или не хватает мест"]
    S6 -->|Да| S7{Это переговорная<br/>или зона столов целиком?}
    S7 -->|Да| S8{Бронируется<br/>всё помещение?}
    S8 -->|Нет| ERR5["Отказ: можно только<br/>забронировать целиком"]
    S8 -->|Да| CALC["Рассчитать стоимость<br/>по тарифу или абонементу"]
    S7 -->|Нет| CALC
    CALC --> SAVE["Записать бронирование<br/>в базу данных"]
    SAVE --> OK["Показать пользователю<br/>подтверждение"]
    ERR1 --> END([Конец])
    ERR2 --> END
    ERR3 --> END
    ERR4 --> END
    ERR5 --> END
    OK --> END
```

### Текст для пояснительной записки

Алгоритм запускается после выбора места на карте и указания даты, времени и числа человек. Система проверяет режим работы коворкинга, корректность даты и времени, доступность места. Запрошенный интервал разбивается на 15-минутные отрезки; для каждого определяется занятость. Если во всех отрезках достаточно свободных мест, рассчитывается стоимость и сохраняется бронь. При нарушении любого условия пользователь получает сообщение с причиной отказа.

## Рисунок 21 — Алгоритм формирования данных интерактивной карты

**Назначение:** подготовить актуальные сведения о местах для отображения на карте.  
**Связь с BPMN:** обновление карты после бронирования и при загрузке страницы.

```mermaid
flowchart TD
    START([Начало]) --> IN["Получить дату и временной интервал,<br/>выбранные пользователем"]
    IN --> LOAD["Загрузить сведения о местах<br/>из базы данных"]
    LOAD --> GEO["Загрузить координаты и размеры<br/>из файла планировки"]
    GEO --> LOOP["Обработать следующее<br/>место на карте"]
    LOOP --> M1{Место на техобслуживании<br/>или служебная зона?}
    M1 -->|Да| GRAY["Пометить как недоступное,<br/>отобразить серым"]
    M1 -->|Нет| M2["Посчитать занятость<br/>на выбранный интервал"]
    M2 --> M3{Никто не бронировал<br/>в этот интервал?}
    M3 -->|Да| GREEN["Пометить как свободное,<br/>отобразить зелёным"]
    M3 -->|Нет| M4{Заняты все места<br/>за столом или в зоне?}
    M4 -->|Да| RED["Пометить как занятое,<br/>отобразить красным"]
    M4 -->|Нет| YELLOW["Пометить как частично занятое,<br/>отобразить жёлтым"]
    GRAY --> MERGE
    GREEN --> MERGE
    RED --> MERGE
    YELLOW --> MERGE["Собрать данные места:<br/>название, код, координаты,<br/>статус и цвет"]
    MERGE --> MORE{Остались<br/>необработанные места?}
    MORE -->|Да| LOOP
    MORE -->|Нет| OUT["Передать данные<br/>для отрисовки карты<br/>в браузере"]
    OUT --> END([Конец])
```

### Текст для пояснительной записки

Алгоритм выполняется при загрузке карты и при смене даты или времени. Система объединяет сведения о местах из базы данных с геометрией из файла планировки. Для каждого объекта определяется занятость: техобслуживание — серый цвет; свободно — зелёный; занята часть мест — жёлтый; всё занято — красный. Результат передаётся для отрисовки карты и списка доступного времени.

## Рисунок 22 — Алгоритм расчёта стоимости бронирования

**Назначение:** вычислить сумму к оплате перед сохранением брони или для показа в форме.  
**Связь с BPMN:** шаг «рассчитать стоимость» процесса «Создание бронирования».

```mermaid
flowchart TD
    START([Начало]) --> IN["Получить: место, тип тарифа,<br/>время начала и окончания,<br/>число человек, дату брони,<br/>клиента"]
    IN --> P1["Загрузить из базы данных<br/>сведения о месте и цене тарифа"]
    P1 --> P2{Для места назначена<br/>категория и нужный тариф?}
    P2 -->|Нет| ERR1["Сообщить: тариф<br/>для этого места недоступен"]
    P2 -->|Да| P3{Это закрытая зона<br/>со столами внутри?}
    P3 -->|Да| P4{Бронируется<br/>вся зона целиком?}
    P4 -->|Нет| ERR2["Сообщить: зону можно<br/>забронировать только целиком"]
    P4 -->|Да| TYPE
    P3 -->|Нет| TYPE["Определить тип объекта:<br/>отдельный стол,<br/>переговорная или зона столов"]
    TYPE --> P5{Какой выбран тариф?}
    P5 -->|Почасовой| P6["Посчитать длительность:<br/>сколько 15-минутных отрезков<br/>между началом и концом"]
    P6 --> P7{Цена берётся<br/>за всё помещение?}
    P7 -->|Да| F1["Сумма = цена за час<br/>умножить на длительность"]
    P7 -->|Нет| F2["Сумма = цена за час<br/>умножить на длительность<br/>и на число человек"]
    P5 -->|Недельный или месячный| P8{Цена берётся<br/>за всё помещение?}
    P8 -->|Да| F3["Сумма = фиксированная<br/>цена тарифа за период"]
    P8 -->|Нет| F4["Сумма = цена тарифа<br/>умножить на число человек"]
    F1 --> ROUND
    F2 --> ROUND
    F3 --> ROUND
    F4 --> ROUND["Округлить сумму<br/>до целых рублей"]
    ROUND --> P9{Почасовой тариф<br/>и указана дата?}
    P9 -->|Нет| OUT1["Вернуть рассчитанную сумму"]
    P9 -->|Да| P10{У клиента есть действующий<br/>абонемент на этот день<br/>и тип места, лимит часов<br/>не исчерпан?}
    P10 -->|Да| OUT2["К оплате — 0 рублей,<br/>полную стоимость сохранить<br/>для справки"]
    P10 -->|Нет| OUT1
    ERR1 --> END([Конец])
    ERR2 --> END
    OUT1 --> END
    OUT2 --> END
```

### Текст для пояснительной записки

Алгоритм запускается после проверки доступности или при изменении параметров в форме бронирования. Система определяет тип объекта — стол, переговорную или зону — и выбирает формулу. Для почасового тарифа длительность считается через 15-минутные отрезки. Для переговорных и зон цена берётся за всё помещение; для столов умножается на число человек. При действующем абонементе сумма к оплате обнуляется.

## Рисунок 23 — Алгоритм определения статуса места на интерактивной карте

**Назначение:** определить, свободно ли место, и каким цветом его показать на карте.  
**Связь с BPMN:** шаг «обновить карту» после бронирования и при загрузке страницы.

```mermaid
flowchart TD
    START([Начало]) --> IN["Получить текущий момент времени<br/>или дату и интервал<br/>для формы бронирования"]
    IN --> L1["Загрузить планировку этажа<br/>и список мест из базы данных"]
    L1 --> L2["Найти все активные брони,<br/>пересекающиеся с интервалом"]
    L2 --> LOOP["Обработать следующее<br/>место на карте"]
    LOOP --> M1{Место переведено<br/>на техобслуживание?}
    M1 -->|Да| GRAY["Статус: недоступно,<br/>цвет — серый"]
    M1 -->|Нет| M2{Это зона<br/>с несколькими столами?}
    M2 -->|Да| M3["Сложить занятость<br/>и вместимость<br/>всех столов внутри зоны"]
    M3 --> M4
    M2 -->|Нет| M5["Посчитать, сколько человек<br/>забронировало это место<br/>в выбранный интервал"]
    M5 --> M4{Никто не бронировал?}
    M4 -->|Да| GREEN["Статус: свободно,<br/>цвет — зелёный"]
    M4 -->|Нет| M6{Занята только часть мест,<br/>а стол или зона<br/>вмещает больше одного?}
    M6 -->|Да| YELLOW["Статус: частично занято,<br/>цвет — жёлтый"]
    M6 -->|Нет| RED["Статус: полностью занято,<br/>цвет — красный"]
    GRAY --> MERGE
    GREEN --> MERGE
    YELLOW --> MERGE
    RED --> MERGE["Добавить к данным места:<br/>название, координаты,<br/>занятость, цвет"]
    MERGE --> MORE{Остались<br/>необработанные места?}
    MORE -->|Да| LOOP
    MORE -->|Нет| OUT["Передать готовые данные<br/>для отрисовки карты<br/>и временной шкалы"]
    OUT --> END([Конец])
```

### Текст для пояснительной записки

Алгоритм выполняется при каждой загрузке карты и при обновлении временной шкалы в форме бронирования. Система объединяет планировку с данными из базы и считает, сколько человек уже забронировало каждое место. Для зон с несколькими столами занятость суммируется по всем столам внутри. По результату место помечается как свободное (зелёный), частично занятое (жёлтый), полностью занятое (красный) или недоступное из‑за техобслуживания (серый).

## Краткое отличие от BPMN

| BPMN «Создание бронирования» | Алгоритм (рис. 20) |
|------------------------------|---------------------|
| Клиент выбирает место на карте | — |
| Клиент нажимает «Забронировать» | — |
| Система проверяет доступность | Разбиение на отрезки, подсчёт занятости, проверка вместимости |
| Система сохраняет бронь | Расчёт стоимости, запись в базу данных |

| BPMN «обновить карту» | Алгоритм (рис. 21, 23) |
|-----------------------|---------------------|
| Система обновляет карту | Загрузка данных, определение статуса каждого места, передача в браузер |

| BPMN «рассчитать стоимость» | Алгоритм (рис. 22) |
|-----------------------------|---------------------|
| Система рассчитывает стоимость | Выбор формулы по типу места и тарифу, проверка абонемента |
